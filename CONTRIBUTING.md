# Contributing

## Branching
- `main` — protected, only via PR
- `feature/<short-name>` — new work
- `fix/<short-name>` — bugfixes

## Commits
Use Conventional Commits (examples):
- `feat(app): add reader shell`
- `fix(api): handle empty token`
- `chore: bootstrap repo`

## Local checks
- Flutter: `dart format . && flutter analyze && flutter test`
- API: `npm test` (placeholder until tests added)
- AI: `pytest` (placeholder until tests added)
